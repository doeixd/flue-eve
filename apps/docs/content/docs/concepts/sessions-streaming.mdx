# Sessions, Runs, and Streaming

## Sessions

A **session** represents a conversation with an agent. Each session has a unique
`sessionId` (v1: `ses_*` format) and a `continuationToken` for resuming the
conversation across turns.

```http
POST /eve/v1/session
Content-Type: application/json

{ "message": "Hello" }
```

Response (202 Accepted):

```json
{
  "sessionId": "ses_abc123",
  "continuationToken": "tok_xyz789"
}
```

## Continuation tokens

The `continuationToken` lets the client resume the session for the next turn:

```http
POST /eve/v1/session/ses_abc123
Content-Type: application/json

{ "message": "What about...", "continuationToken": "tok_xyz789" }
```

**Token behavior (v1):**

- The **same** `continuationToken` is returned every turn â€” stable per session
- A **stale** token (client sends an old token while a turn is still active) â†’ **409 Conflict**
- A **terminal** session (agent completed, no more turns possible) â†’ **410 Gone**
- Future versions may implement per-turn rotation; v1 keeps it simple

## Streaming

The stream endpoint returns NDJSON (`application/x-ndjson; charset=utf-8`),
one JSON event per line:

```http
GET /eve/v1/session/ses_abc123/stream
```

Response headers:

```
x-eve-stream-format: ndjson
x-eve-stream-version: 16
```

### Reconnect with startIndex

The `startIndex` parameter replays from a known position after a dropped
connection:

```http
GET /eve/v1/session/ses_abc123/stream?startIndex=42
```

The server replays events from the **event journal** starting at index 42,
then continues with live events. The journal is the sole owner of `streamIndex`
â€” never using Flue Durable Streams offsets directly.

**Invalid `startIndex`** values (negative, NaN) â†’ **400 Bad Request**.
