# Getting Started

flue-eve lets you use Eve's HTTP contract, client SDK, and React hooks on top
of Flue's open runtime. Whether you're starting fresh, migrating from Eve, or
just need the client library, pick your path below.

## Prerequisites

- **Node.js** 22.19.0 or later (Flue requirement)
- **pnpm** 9.x (or npm/yarn)
- A package manager and a terminal

## What you'll build

At the end of each path, you'll have a working project where:

- Your browser UI at `http://localhost:5173` talks to `/eve/v1/*` — no CORS,
  no external config
- An agent responds to messages with streaming NDJSON events
- You can use `useEveAgent` in React or the `Client` SDK directly
- Mock admission gives you deterministic responses without needing an LLM API key

To test with a real LLM, start `flue dev` and set `FLUE_BASE_URL=http://127.0.0.1:3583`.

## How it works (30-second overview)

```text
Your browser code
  → /eve/v1/session, /stream  (Eve HTTP contract)
  → @flue-eve/compat-server   (translates Eve ↔ Flue)
  → Flue agent instance        (runs the LLM + tools)
```

The [architecture](/docs/concepts/architecture) concept page explains each layer
in detail. The short version: the Vite plugin proxies `/eve/v1/*` to a Flue
server, and the compat-server translates Eve's NDJSON streaming format into
Flue's agent protocol and back.
